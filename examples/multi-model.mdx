Laser supports multiple AI model providers through the `kosong` framework. You can switch between OpenAI, Anthropic, Google, and OpenAI-compatible endpoints.

## Supported Providers

Laser comes configured with these providers:

| Provider | Prefix | Example Model | API Key Env Var |
|----------|--------|---------------|----------------|
| OpenAI | `openai/` | `openai/gpt-5.2-codex` | `OPENAI_API_KEY` |
| Anthropic | `anthropic/` | `anthropic/claude-opus-4-5` | `ANTHROPIC_API_KEY` |
| Google | `google/` | `google/gemini-3-pro-preview` | `GOOGLE_API_KEY` |
| OpenAI-compatible | `chat/` | `chat/glm-4.7` | `OPENAI_API_KEY` + `OPENAI_BASE_URL` |

## Switching Models

### At Startup

Use the `--model` flag with the format `<provider>/<model>`:

```bash
# Use Anthropic's Claude
laser --model anthropic/claude-opus-4-5
```

```bash
# Use Google's Gemini
laser --model google/gemini-3-pro-preview
```

```bash
# Use OpenAI (default)
laser --model openai/gpt-5.2-codex
```

### Setting API Keys

Before using a provider, export the required API key:

<CodeGroup>
```bash OpenAI
export OPENAI_API_KEY="sk-..."
laser --model openai/gpt-5.2-codex
```

```bash Anthropic
export ANTHROPIC_API_KEY="sk-ant-..."
laser --model anthropic/claude-opus-4-5
```

```bash Google
export GOOGLE_API_KEY="AIza..."
laser --model google/gemini-3-pro-preview
```
</CodeGroup>

## OpenAI-Compatible Endpoints

The `chat/*` provider supports any OpenAI-compatible API. This is useful for:

- Local models (via LM Studio, Ollama with OpenAI compatibility, etc.)
- Custom endpoints
- Alternative providers with OpenAI-compatible APIs

### Example: Local LM Studio

```bash
export OPENAI_BASE_URL="http://localhost:1234/v1"
export OPENAI_API_KEY="not-needed"
laser --model chat/local-model
```

### Example: Custom Endpoint

```bash
export OPENAI_BASE_URL="https://api.example.com/v1"
export OPENAI_API_KEY="your-key"
laser --model chat/glm-4.7
```

## Controlling Token Limits

For Anthropic models, you can cap the maximum tokens per response:

```bash
laser --model anthropic/claude-opus-4-5 --max-tokens 4096
```

<Note>
The `--max-tokens` flag only affects Anthropic models. Other providers ignore this setting and use their own defaults.
</Note>

## Choosing the Right Model

<AccordionGroup>
  <Accordion title="For complex refactoring" icon="arrows-spin">
    Use larger models with strong reasoning:
    - `anthropic/claude-opus-4-5`
    - `openai/gpt-5.2-codex`
    
    These models handle multi-file changes and architectural decisions better.
  </Accordion>

  <Accordion title="For quick fixes" icon="screwdriver-wrench">
    Smaller, faster models work well:
    - `google/gemini-3-pro-preview`
    - `anthropic/claude-sonnet-4`
    
    Lower latency and cost for simple tasks.
  </Accordion>

  <Accordion title="For privacy-sensitive code" icon="shield">
    Use local models via OpenAI-compatible endpoints:
    - `chat/local-model` (via LM Studio, Ollama, etc.)
    
    Your code never leaves your machine.
  </Accordion>

  <Accordion title="For cost optimization" icon="dollar-sign">
    Monitor token usage with `/usage` and:
    - Use `/reset` to clear history between tasks
    - Choose smaller models when appropriate
    - Leverage prompt caching (Anthropic)
  </Accordion>
</AccordionGroup>

## Example: Switching Mid-Project

You can restart Laser with a different model at any time:

<Steps>
  <Step title="Start with a fast model for exploration">
    ```bash
    laser --model google/gemini-3-pro-preview
    ```
    
    Use this to quickly explore the codebase and identify the files you need to change.
  </Step>

  <Step title="Exit and switch to a stronger model">
    ```text
    laser> /quit
    ```
    
    Then restart with a more capable model:
    
    ```bash
    laser --model anthropic/claude-opus-4-5
    ```
  </Step>

  <Step title="Complete the complex refactoring">
    The stronger model can handle architectural changes and multi-file refactoring.
  </Step>
</Steps>

## Provider-Specific Features

### Anthropic: Prompt Caching

Anthropic models automatically cache repeated content (like file contents and conversation history). You'll see cache statistics with `/usage`:

```text
Token usage (cumulative)

Input : 125000  (other=45000, cache_read=60000, cache_create=20000, cached=64.0%)
Output:  12000

Total : 137000
```

Higher cache percentages mean lower costs on subsequent requests.

### OpenAI: Streaming Responses

OpenAI models stream responses token-by-token for faster perceived performance.

## Troubleshooting

<Warning>
If you see "Unknown provider" errors, check that your `--model` flag uses the correct format: `<provider>/<model>`

Valid providers: `openai`, `anthropic`, `google`, `chat`
</Warning>

### Missing API Keys

If Laser exits with authentication errors:

1. Verify the environment variable is set: `echo $OPENAI_API_KEY`
2. Check for typos in the variable name
3. Ensure the key has sufficient permissions/credits

### Model Not Found

If the provider returns "model not found":

1. Verify the model name is correct and available
2. Check that your API key has access to that model
3. For `chat/*` provider, ensure `OPENAI_BASE_URL` points to a valid endpoint
