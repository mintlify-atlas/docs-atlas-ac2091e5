Laser supports multiple AI model providers through the [`kosong`](https://pypi.org/project/kosong/) library. Select a provider using the `--model` flag with the format `<provider>/<model>`.

## Supported Providers

The following providers are available in Laser:

### anthropic

<ParamField path="anthropic" type="provider">
  Anthropic Claude models
  
  **Provider class:** `Anthropic` (from `kosong.contrib.chat_provider.anthropic`)
  
  **Environment variable:** `ANTHROPIC_API_KEY`
  
  **Example models:**
  ```bash
  laser --model anthropic/claude-opus-4-5
  laser --model anthropic/claude-sonnet-4
  laser --model anthropic/claude-3-5-sonnet-20241022
  ```
  
  **Special notes:**
  - The `--max-tokens` flag applies to Anthropic models
  - Supports prompt caching for cost savings on repeated context
</ParamField>

### google

<ParamField path="google" type="provider">
  Google Gemini models
  
  **Provider class:** `GoogleGenAI` (from `kosong.contrib.chat_provider.google_genai`)
  
  **Environment variable:** `GOOGLE_API_KEY`
  
  **Example models:**
  ```bash
  laser --model google/gemini-3-pro-preview
  laser --model google/gemini-2.0-flash-exp
  laser --model google/gemini-1.5-pro
  ```
</ParamField>

### openai

<ParamField path="openai" type="provider">
  OpenAI models with responses API
  
  **Provider class:** `OpenAIResponses` (from `kosong.contrib.chat_provider.openai_responses`)
  
  **Environment variable:** `OPENAI_API_KEY`
  
  **Example models:**
  ```bash
  laser --model openai/gpt-5.2-codex
  laser --model openai/gpt-4o
  laser --model openai/gpt-4-turbo
  ```
  
  **Default model:** `openai/gpt-5.2-codex`
</ParamField>

### chat

<ParamField path="chat" type="provider">
  OpenAI-compatible legacy chat models
  
  **Provider class:** `OpenAILegacy` (from `kosong.contrib.chat_provider.openai_legacy`)
  
  **Environment variables:**
  - `OPENAI_API_KEY` (required)
  - `OPENAI_BASE_URL` (optional, for non-OpenAI endpoints)
  
  **Example models:**
  ```bash
  laser --model chat/glm-4.7
  laser --model chat/gpt-3.5-turbo
  ```
  
  **Use cases:**
  - OpenAI-compatible local models
  - Third-party providers with OpenAI-compatible APIs
  - Older OpenAI models using the legacy format
  
  Set `OPENAI_BASE_URL` to point to custom endpoints:
  ```bash
  export OPENAI_BASE_URL="http://localhost:8080/v1"
  laser --model chat/custom-model
  ```
</ParamField>

## Configuration

Provider configurations are defined in `laser/main.py`:

```python
PROVIDER_CONFIGS = {
    "anthropic": (Anthropic, {}),
    "google": (GoogleGenAI, {}),
    "openai": (OpenAIResponses, {}),
    "chat": (OpenAILegacy, {}),
}
```

## Setting API Keys

Before using a provider, set the required environment variable:

```bash
# Anthropic
export ANTHROPIC_API_KEY="your-key-here"

# Google
export GOOGLE_API_KEY="your-key-here"

# OpenAI
export OPENAI_API_KEY="your-key-here"
```

Then run Laser with your chosen model:

```bash
laser --model anthropic/claude-opus-4-5
```

## Model Selection Examples

### Use Claude Opus for complex reasoning
```bash
laser --model anthropic/claude-opus-4-5 --max-tokens 4096
```

### Use Gemini for cost-effective tasks
```bash
laser --model google/gemini-2.0-flash-exp
```

### Use OpenAI's latest models
```bash
laser --model openai/gpt-5.2-codex
```

### Use local or third-party compatible models
```bash
export OPENAI_BASE_URL="http://localhost:8080/v1"
laser --model chat/local-model
```

## Error Handling

If you specify an unknown provider, Laser will exit with an error:

```bash
$ laser --model unknown/model-name
Unknown provider 'unknown'. Known providers: anthropic, chat, google, openai
```

Make sure your `--model` argument uses the format `<provider>/<model>` where the provider is one of: `anthropic`, `google`, `openai`, or `chat`.
