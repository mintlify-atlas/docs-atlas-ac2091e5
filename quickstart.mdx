## Starting Laser

Navigate to your repository and start Laser:

```bash
laser
```

You'll see a welcome banner with your current model and directory:

```
╭─────────────────────────────────────────╮
│ Laser                                   │
│ - Model: openai/gpt-5.2-codex          │
│ - Directory: /path/to/your/repo        │
│ - Tip: type /quit to exit              │
│        type /reset to clear history    │
╰─────────────────────────────────────────╯
```

## Basic Workflow

<Steps>
  <Step title="Start Laser in your repository">
    ```bash
    cd /path/to/your/repo
    laser
    ```
  </Step>
  
  <Step title="Describe what you want to change">
    Tell Laser what you need in plain language:
    
    ```
    Add error handling to the user authentication function
    ```
  </Step>
  
  <Step title="Review and iterate">
    Laser will:
    - Inspect your repository
    - Make the necessary edits
    - Run commands to verify changes
    - Show you diffs
    
    You can continue the conversation to refine the changes.
  </Step>
</Steps>

## Command Line Options

Laser supports several useful flags:

<CodeGroup>

```bash Choose a Model
laser --model openai/gpt-5.2-codex
laser --model anthropic/claude-opus-4-5
laser --model google/gemini-3-pro-preview
```

```bash Change Directory
laser --cwd /path/to/your/repo
```

```bash Non-Interactive Mode
laser --prompt "Run tests and summarize failures"
```

```bash Planning Mode
laser --plan
```

```bash Max Tokens (Anthropic only)
laser --max-tokens 4096
```

</CodeGroup>

<Note>
  The `--cwd` flag changes the working directory where Laser operates (reads/writes files and runs commands).
</Note>

## Interactive Commands

While Laser is running, you can use these slash commands:

| Command | Description |
|---------|-------------|
| `/usage` | Print cumulative token usage for the current session |
| `/reset` | Clear the current chat history and start fresh |
| `/quit` | Exit Laser |

### Running Local Shell Commands

You can run shell commands directly by prefixing with `!`:

```bash
!pytest -q
!git status
!npm run build
```

The command runs in your current working directory and prints stdout/stderr.

## Multi-Line Input

For longer requests, use a trailing backslash (`\`) to continue to the next line:

```text
Write a script that:\
- scans all Python files\
- prints unused imports\
- and suggests fixes
```

Laser will continue reading lines until you send one without a trailing backslash.

## Planning Mode

When you use the `--plan` flag, Laser writes a checklist plan file before taking actions:

```bash
laser --plan
```

Laser will:

1. Create a `plans/` directory if needed
2. Write a Markdown checklist to `plans/<task_slug>_plan.md`
3. Update the checklist as it works (checking items off, adding/removing items)
4. Proceed with the task

<Tip>
  Planning mode is useful for complex tasks where you want visibility into Laser's step-by-step approach.
</Tip>

## Example Session

Here's a typical interactive session:

```bash
# Start Laser
$ laser

 laser > Add type hints to the database module

# Laser inspects files, makes edits, and shows results

 laser > Run the type checker to verify

# Laser runs mypy or pyright

 laser > /usage

# Shows token usage statistics

 laser > /quit
```

## Working with AGENTS.md

If your repository has an `AGENTS.md` file at the root, Laser will read it at the start of each task and follow those instructions.

Example `AGENTS.md`:

```markdown
## response style
just respond like you are TARS from the movie Interstellar

## formatting
run `uvx ruff format <file>` after editing python files
```

<Warning>
  If `AGENTS.md` conflicts with Laser's built-in safety rules (like avoiding destructive commands), Laser's rules take priority.
</Warning>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Shell commands fail">
    Laser surfaces stderr from failed commands. Try:
    
    - Re-running the exact command manually
    - Checking that required tools are installed
    - Ensuring you're in the correct directory (use `--cwd`)
  </Accordion>
  
  <Accordion title="Agent operates in wrong directory">
    Use the `--cwd` flag to specify the correct directory:
    
    ```bash
    laser --cwd /path/to/repo
    ```
  </Accordion>
  
  <Accordion title="Model provider errors">
    Ensure you've set the correct API key environment variable:
    
    - OpenAI: `OPENAI_API_KEY`
    - Anthropic: `ANTHROPIC_API_KEY`
    - Google: `GOOGLE_API_KEY`
  </Accordion>
</AccordionGroup>
